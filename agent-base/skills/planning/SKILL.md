---
name: planning
description: A reusable planning workflow for breaking tasks into milestones, files, and validation steps.
---

Use this skill when starting new implementation work.

Steps:

1. restate the request clearly
2. identify the main goal and constraints
3. split the work into 3–5 milestones
4. name files or components to create/update
5. define verification steps (tests, review, examples)
