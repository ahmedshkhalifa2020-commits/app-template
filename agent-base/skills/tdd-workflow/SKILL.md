---
name: tdd-workflow
description: A test-driven development workflow for implementing features and fixes.
---

Use this skill to implement code changes with tests first.

Steps:

1. define the failing behavior or bug as a test case
2. create or update the test file before application code
3. run the test and confirm it fails for the right reason
4. implement the minimal code change to satisfy the test
5. rerun the test and ensure it passes
6. refactor for clarity and maintainability
