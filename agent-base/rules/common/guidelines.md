# Common Development Guidelines

This document contains simple always-follow rules for project work.

## Goals

- keep planning explicit
- write tests for behavior before implementation
- prefer small, clear changes
- validate with review and examples

## Guidelines

- write a task plan before editing code
- make tests the first concrete output of a change
- keep variable and function names descriptive
- avoid large, unreviewed refactors in a single step
- document the workflow or decisions when needed
